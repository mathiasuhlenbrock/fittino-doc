class Fittino::AnalysisTool¶

Abstract class which provides basic functionality for analysis tools like optimizers or samplers.

This class contains auxiliary objects needed by every analysis tool such as the tool’s name, a pointer to the model to be analysed, a counter for the number of iteration steps and a random number generator.

Inherits from Fittino::Tool

Subclassed by Fittino::OptimizerBase, Fittino::SamplerBase

class Fittino::Controller¶

Singleton class for controlling the execution flow of Fittino.

The instance of the Controller class is the first object that is created at the beginning of the execution of Fittino. Depending on the user configuration the controller creates instances of further objects (either directly or with the help of a factory) and advises them to perform the specified tasks, hereby controlling the program’s overall execution flow.

The controller adresses its purpose in three distinct phases: At first, the controller initializes Fittino by letting an input file interpreter read in user specified options. After that further objects are created and put into action according to the user’s configuration. At the end, the controller provides the controlled termination of Fittino.

class Fittino::ExceptionBase¶

Fittino exception base class.

Exceptions make up the backbone of the error handling system in Fittino. This class provides basic functionality for the various Fittino exception classes. Their main purpose is to print out messages with further information about the occurred exception in a uniform way. Supported Fittino exceptions are

  • CalculatorException
  • ConfigurationException
  • ExecutorException
  • InputException
  • OptimizerException
  • RedirectorException
  • SLHAFileException
  • TimeoutExecutorException

Subclassed by Fittino::CalculatorException, Fittino::ConfigurationException, Fittino::ExecutorException, Fittino::InputException, Fittino::LogicException, Fittino::OptimizerException, Fittino::RedirectorException, Fittino::SLHAFileException

Public Functions
